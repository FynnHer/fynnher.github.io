Why I'm Learning Rust

By Fynn-Lasse Herrmann

After spending most of my coding time with Python and Dart, I've recently embarked on a new adventure: learning Rust. In this post, I'll share why I chose Rust and what I'm hoping to achieve with it.

The Appeal of Systems Programming

As someone who's primarily worked with high-level languages, I've always been curious about what happens "under the hood." How does memory actually work? What makes a program fast or slow at the fundamental level? How are operating systems built?

These questions led me to explore systems programming, and specifically to Rust.

Why Rust?

I considered several options for diving into systems programming, including C and C++. Here's why I ultimately chose Rust:

1. Memory Safety Without Garbage Collection

Rust's ownership system prevents common bugs like null pointer dereferences, buffer overflows, and data races at compile time. This is huge for someone coming from managed languages!

// The compiler catches this error
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1 is moved to s2
    // println!("{}", s1);  // This would cause a compile error!
    println!("{}", s2);  // This works fine
}

2. Modern Tooling

Coming from Python and Dart, I'm used to good tooling. Rust doesn't disappoint:

  • Cargo - Package manager and build tool
  • rustfmt - Automatic code formatting
  • clippy - Linting with helpful suggestions
  • rust-analyzer - Excellent IDE support

3. Performance

Rust gives you C-like performance without the footguns. It's used in performance-critical applications from web browsers (Firefox, Chrome parts) to databases.

4. Growing Ecosystem

While not as vast as Python's, Rust's ecosystem is growing rapidly. There are crates (packages) for everything from web development to embedded systems.

My Learning Journey

Here's how I've been approaching learning Rust:

  1. The Rust Book - The official guide is excellent and free
  2. Rustlings - Small exercises that help you learn by doing
  3. Small Projects - Building CLI tools to practice
  4. Exploring OS Dev - Looking into kernel development resources

The Challenge

I won't sugarcoat it: Rust has a steep learning curve. The borrow checker can be frustrating at first, and concepts like lifetimes take time to internalize.

// Lifetimes can be confusing at first
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

But the compiler errors are incredibly helpful. Rust tells you not just what's wrong, but often how to fix it!

What I Want to Build

My goals with Rust include:

  • Kernel Experiments - Understanding how operating systems work
  • CLI Tools - Fast, reliable command-line utilities
  • Performance-Critical Components - Parts of larger systems where speed matters
  • WebAssembly - Rust compiles to WASM, opening up browser possibilities

Resources I Recommend

If you're interested in learning Rust, here are some resources:

  • The Rust Programming Language Book - Free and comprehensive
  • Rustlings - Interactive exercises
  • Writing an OS in Rust - Amazing tutorial series

Final Thoughts

Learning Rust is challenging but incredibly rewarding. It's making me a better programmer by forcing me to think more carefully about memory, ownership, and program structure.

"In Rust, we don't just write code. We have a conversation with the compiler."
